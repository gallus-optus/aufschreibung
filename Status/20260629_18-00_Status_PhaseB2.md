### Wichtigste implementierte Funktionen

**IndexedDB (app.js, Abschnitt 2):**
- 8 Object Stores: meta, auth, lebensmittel, eigengerichte, zutaten, konfiguration, erfassung, vitaldaten
- Promise-basierte Hilfsfunktionen: dbLesen, dbSchreiben, dbLoeschen, dbAllesLesen, dbZaehlen, dbAllesLoeschen, dbMassenSchreiben, dbIndexLesen
- Index `nach_gericht_id` auf dem zutaten-Store für schnelle Zutat-Abfragen

**Dropbox OAuth PKCE (app.js, Abschnitt 3):**
- PKCE Code Verifier + Code Challenge mit SHA-256 (Web Crypto API)
- token_access_type=offline → Refresh Token wird mitgespeichert
- Automatische Token-Erneuerung bei 401-Antworten
- State-Prüfung gegen CSRF

**CSV-Parser (app.js, Abschnitt 5):**
- Semikolon-Trennzeichen, Komma als Dezimalzeichen
- UTF-8 BOM wird automatisch entfernt
- Funktion `deutscheZahlParsen()` für alle Nährwert-Felder

**Nährwert-Berechnung für Eigengerichte:**
- Einmalig beim Import berechnet, gecacht in IndexedDB (kcal_pro_100g_berechnet)
- Felder: kcal, eiweiss, kh, fett, salz jeweils pro 100g
- Bei UNBEKANNT-Zutaten: kcal_unvollstaendig=true, Zähler anzahl_unbekannte_zutaten
- In der Gerichte-Liste "*" hinter kcal-Wert wenn unvollständig

**Routing (app.js, Abschnitt 6):**
- Hash-basiert: #/lebensmittel, #/gerichte, #/einstellungen,
  #/lebensmittel-detail/LM_0001, #/gericht/EG_013
- Browser-Zurück-Knopf funktioniert über history.back()
- Tab-Leiste nur bei den drei Haupt-Bildschirmen sichtbar

## Architektur-Entscheidungen während des Bauens

**Gerichte-Chips:** Werden aus dem gericht_kategorie-Feld der IndexedDB gebaut,
nicht aus 05_Konfiguration.json. Grund: einfacher und setzt kein komplettes
Konfigurations-Schema voraus.

**Lebensmittel-Cache:** lebensmittelCache und gerichteCache sind JS-Variablen
im Modul-Scope. Werden beim Abmelden und beim Sync geleert. Kein Re-Lesen aus
IndexedDB solange die App offen ist → schnelle Filter ohne DB-Overhead.

**Token-Ablaufzeit:** Access Token gilt 4 Stunden. Wenn weniger als 5 Minuten
Restlaufzeit, wird vor dem nächsten API-Aufruf automatisch erneuert.
Die Ablaufzeit wird als Unix-Timestamp (ms) gespeichert.

**escapeHtml():** Alle Daten aus der IndexedDB, die per innerHTML in den DOM
geschrieben werden, gehen durch escapeHtml(). Schützt gegen unerwartete
HTML/JS-Injektion aus den CSV-Feldern.

**Icons:** Mit PowerShell System.Drawing erzeugt (kein Python/Node verfügbar).
Blauer Hintergrund #0061FF, weißes "A" (Arial Bold). Können in späteren
Phasen durch ein professionelles Icon ersetzt werden.

## Bekannte Einschränkungen

- **Lokaler Test nicht möglich:** Kein npx/Node auf dem Entwicklungsrechner.
  Test nur auf GitHub Pages möglich. Die OAuth-Redirect-URI ist auf die
  GitHub-Pages-URL eingeschränkt — lokale Tests werden nach Login auf
  github.io weitergeleitet, was vom lokalen Server nicht funktioniert.

- **Service Worker auf GitHub Pages:** Bei erstem Deployment kann der
  Service Worker bis zu einer Seite-Aktualisierung inaktiv sein. Beim
  zweiten Öffnen ist Offline-Modus aktiv.

- **Dropbox SDK von CDN:** Die Datei
  `https://cdn.jsdelivr.net/npm/dropbox@10.34.0/dist/Dropbox-sdk.min.js`
  wird per `<script>`-Tag geladen, aber die App nutzt ihn aktuell NICHT
  direkt (der PKCE-Flow ist selbst implementiert). Das SDK ist für spätere
  Phasen (Schreiben zurück nach Dropbox) bereits eingebunden.

- **Konfigurations-Datei:** 05_Konfiguration.json wird heruntergeladen und
  in IndexedDB gespeichert, aber in Phase B2 noch nicht für die UI genutzt
  (Zeitregeln, Gruppen-Liste etc. kommen in späteren Phasen).

## Nachtrag 2026-06-30 — Mockup-Abgleich und Fehlerbehebung

Nach Lieferung des Mockups (`Mockups/phase_b2_mockup.html`) wurde der
Code dagegen abgeglichen und überarbeitet. Geänderte Dateien:
`index.html`, `style.css`, `app.js`.

### Behobene Fehler (Funktion)

1. **Navigations-Hänger nach Erst-Import:** `navigieren()` setzte nur
   `location.hash`. War der Ziel-Hash schon der aktuelle (z. B. nach dem
   OAuth-Redirect `#/lebensmittel`), feuerte kein `hashchange` und die App
   blieb auf dem Import-Bildschirm hängen. Jetzt wird in diesem Fall direkt
   neu gerendert.

2. **Doppelter Scroll-Container in den Detail-Ansichten:** Es war ein
   `.ansicht-inhalt` in einem `.ansicht-inhalt` verschachtelt (doppeltes
   Padding, Hero nicht randlos). Neue Struktur: `.detail-scroll` (ohne
   Padding) + randloser Hero + gepolstertes `.detail-body`.

3. **Dropbox-API ohne Argumente:** Bei `users/get_current_account` und
   `users/get_space_usage` wurde fälschlich `Content-Type: application/json`
   mit Body `null` gesendet, was Dropbox mit einem JSON-Decodier-Fehler
   quittieren kann. Jetzt wird Content-Type/Body nur bei echten Argumenten
   gesetzt.

4. **Deutsche Dezimalkommas:** Nährwerte wurden mit Punkt ausgegeben
   ("9.1 g"). Jetzt durchgängig Komma ("9,1 g") über die neue Hilfsfunktion
   `zahlDe()`. kcal werden für die Anzeige gerundet.

5. Toter Code entfernt (`routerHistorie`), kleinere Aufräumarbeiten.
